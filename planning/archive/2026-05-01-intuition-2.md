## Position Prediction Baselines (Question 1)

**Lateral velocity prediction, Random Forest vs persistence baseline:**

| Horizon | Persistence MAE | RF MAE | RF R² | ML Improvement |
|---------|----------------|--------|-------|----------------|
| N=5 ticks | 2.786 px/tick | 2.614 | 0.640 | 6.2% |
| N=10 ticks | 4.304 px/tick | 3.861 | 0.322 | 10.3% |
| N=20 ticks | 5.983 px/tick | 4.585 | 0.084 | **23.4%** |

**Key finding:** ML advantage grows with prediction horizon. At 20 ticks ahead, RF reduces error by 23% over naive persistence. This is exactly the regime that matters for targeting — bullet travel time at medium range is 15-25 ticks.

**Top features for N=10 prediction:**
1. `opponent_lateral_direction` (35%) — which way they're currently orbiting
2. `opponent_is_decelerating` (12%) — braking signals upcoming direction change
3. `opponent_velocity` (10%) — current speed
4. `opponent_lateral_velocity` (9%) — current lateral speed (the thing we're predicting)
5. `opponent_time_since_direction_change` (7%) — how long since last reversal

**R² drops from 0.64 (N=5) to 0.08 (N=20)** — at 20 ticks, the model barely explains variance. This matches competitive Robocode experience: beyond ~15 ticks, movement becomes essentially unpredictable for top bots. Pattern matching or GF statistics are needed, not point prediction.

## Implications for ML

1. **Short-horizon prediction (N≤10) is tractable** — RF gives meaningful improvement. Worth pursuing with richer features and sequence models.
2. **Long-horizon prediction (N≥20) needs a different approach** — point prediction fails. GuessFactor distribution estimation (as competitive bots do) is more appropriate.
3. **Bot identification is critical** — if we can identify BeepBoop/DrussGT early, we know their movement is smooth and deliberate (predict differently). For ScalarR, expect stop-and-go.
4. **Opponent lateral direction is the #1 feature** — matches competitive wisdom. Add more direction-change-related features (reversal frequency, segment length).
5. **Adaptation detection matters** — against DrussGT, our model should weight recent rounds more heavily. Against BeepBoop, early data is as good as late data.
